# Automated Review Synthesis: Wildberries Intelligence

Данный проект представляет собой программную систему для автоматизированного сбора и интеллектуального анализа пользовательских отзывов с маркетплейса Wildberries. Система трансформирует массивы неструктурированных текстовых данных в лаконичное резюме, выделяя ключевые достоинства и недостатки товара.

## Описание проблемы и предлагаемого решения

В современной электронной коммерции потребители сталкиваются с проблемой информационной перегрузки. Согласно исследованиям, более 93% покупателей изучают отзывы перед покупкой, однако среднестатистический пользователь способен просмотреть не более 10–15 записей. Это ведет к принятию решений на основе нерепрезентативной выборки.

Проект автоматизирует переход от ручного анализа (занимающего 10–20 минут на один товар) к интеллектуальной агрегации с помощью больших языковых моделей. Время обработки одного запроса составляет около 30 секунд, что обеспечивает 95% экономии времени при сохранении высокой объективности выводов.

## Технологический стек

Система разработана на языке **Java 17** с использованием системы сборки **Maven 3**. Архитектура минимизирует количество внешних зависимостей для обеспечения стабильности и производительности:

*   **Сбор данных:** Playwright for Java 1.47. Используется для обхода динамического рендеринга (SPA) и имитации поведения реального пользователя, что позволяет эффективно преодолевать системы защиты маркетплейса.
*   **Анализ текста:** Llama 3.3 70B через Groq API. Модель применяется в режиме Zero-shot inference, обеспечивая высокое качество обработки русского языка без необходимости обучения на специфических датасетах.
*   **Бэкенд:** Встроенные средства JDK (HttpClient и HttpServer) и библиотека Gson для сериализации данных.
*   **Интерфейсы:** Веб-приложение на Vanilla JS и Telegram-бот с механизмом бесшовной синхронизации данных через 6-символьные коды.

## Архитектура системы

Процесс обработки данных включает пять основных этапов:
1. **Ввод:** Прием артикула товара через веб-интерфейс или мессенджер.
2. **Парсинг:** Автоматическая навигация и извлечение не менее 30 последних отзывов.
3. **Синтез:** Генерация структурированного JSON-ответа (3-5 плюсов, 3-5 минусов и итоговое резюме).
4. **Хранение:** Сохранение истории запросов в локальный файл `users.json`.
5. **Выдача:** Отображение агрегированного результата пользователю.

## Метрики качества

В ходе испытаний системы на реальных товарах были получены следующие показатели качества извлечения информации:
*   **Precision (Точность):** 0.83
*   **Recall (Полнота):** 0.80
*   **F1-мера:** 0.81

Среднее время выполнения полного цикла запроса составляет 31.6 секунды, что полностью соответствует требованиям к производительности.

## Требования к окружению

Для развертывания и запуска системы необходимы:
*   Операционная система Windows 10/11.
*   Установленная среда JRE 17 или выше.
*   Браузер Google Chrome (для работы Playwright).
*   Действующий API-ключ Groq и токен Telegram-бота (конфигурируются в файле .env).
